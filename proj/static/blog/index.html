Solution

Plan-O-Gram

I was tasked with creating an application that would produce a simple plan-o-gram (a layout used for setup) of items optimally positioned on shelves given various product and shelf dimensionions. The product dimensionions were stored in a data table while shelf levels and dimensions where taken in the form of user input(this would vary as desired by clients). The plan-o-gram would then be used as a guide for the initial setup of products at the client's location. Since the application only needed to be accessible in-network to a few individuals within a small department, I developed the application in PowerShell as this was readily available in their desktop environment, required no additional setup, and would therefore help deliver their application quickly. The program asks the user to point to a file containing a clients desired products (which typically ranges from hundreds to thousands), then takes the client's desired number of shelf levels and their dimensionions. Given those sets of data, it outputs a plan-o-gram.

This application exemplifies that fact that there are many ways to go about developing any particular solution. It utilizes a PowerShell console and scripts, including a number of methods from the .Net library, and outputs data which is then formatted into an Excel Workbook using Visual Basic. These are a set of tools not often used together. View a completed plan-o-gram.



Program Outline

  • Takes user input and products file.
  • Sets products on shelves optimally given product and shelf dimensions.
  • Outputs data for plan-o-glam.
  • Places and styles data in an Excel workbook.
  • Plan-o-gram within Excel workbook is used as needed.
