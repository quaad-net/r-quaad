Plan-O-Gram

I was tasked with creating an application that would produce a simple plan-o-gram (a layout used for setup) of items optimally positioned on shelves given various product and shelf dimensions. The product dimensions were stored in a data table while shelf levels and dimensions where taken in the form of user input(this would vary as desired by clients). The plan-o-gram would then be used as a guide for the initial setup of products at the client's location. Since the application only needed to be accessible in-network to a few individuals within a small department, I developed the application with PowerShell as this was readily available in their desktop environment, required no additional setup, and would therefore help deliver their application quickly. The program asks the user to point to a file containing a clients desired products (which typically ranges from hundreds to thousands), then also takes the client's shelf dimensions. Given those sets of data, it outputs the plan-o-gram.

This application exemplifies that fact that there are many ways to go about developing any particular solution. It utilizes a PowerShell console and scripts, including a number of methods from the .Net library, and outputs data which is then formatted into an Excel Workbook with Visual Basic.



Outline

  • Application takes user input for dimensions and products file.
  • Sets products on shelves optimally given product and shelf dimensions.
  • Outputs data for plan-o-gram.
  • Places and styles finished plan-o-gram in an Excel workbook.
  • View a completed plan-o-gram.

SQL Views

SQL views are basically functions which operate on data stored in the database to produce resulting datasets that meet some need. I find it particularly useful for analyzing data by filtering larger sets of data on certain conditions and/or by combining data from various tables that meet those conditions. Though the view function is stored in the database, the resulting datasets are not. This involves a trade-off between better performance (stored database tables) vs less storage needs (views). However, issues with performance, in my experience, aren't very noticeable unless the datasets are extremely large.

These SQL views are associated with the data dashboard project above. Also included are the original data files which help give an idea of the operations being accomplished by the views.



Outline

  • The SQL views provide a quick and efficient way to pre-process data within the server before it gets sent over to be handled on the front-end by JavaScript where it gets processed for charting in the application.
  • SQL views for data dashboard.
    • Original data files fetched from various sources pre-database operations.
