Plan-O-Gram

I was tasked with creating an application that would produce a simple plan-o-gram (a visual layout) of items optimally positioned on shelves given various product and shelf dimensions. The product dimensions were stored in a data table while shelf levels and dimensions where taken in the form of user input(this would vary as desired by clients). The plan-o-gram could then be used as a guide for the initial setup of products at the client's location.

The Powershell application asks the user to point to a file containing a clients desired products (which typically ranges from hundreds to thousands), then also takes the client's shelf dimensions. Given those sets of data, it outputs the plan-o-gram.



Outline

  • Application takes user input for dimensions and products file.
  • Sets products on shelves optimally given product and shelf dimensions.
  • Outputs data for plan-o-gram.
  • Places and styles finished plan-o-gram in an Excel workbook.
  • Sample plan-o-gram

Data Dashboard

A work in progress of a web dashboard utilizing some data published by various departments of the U.S. Government. The front-end is written in pure JavaScript, HTML, and CSS, and the back-end in Python. Data is stored in an Azure SQL database and fetched through SQL and Python view functions.

Additional components to come.



Outline

  • Database views are utilized to format data and to make it easier to work with.
